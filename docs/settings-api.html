API & Auth Settings

Expose a threads API, choose a token strategy, and publish reference docs for API consumers.

The API layer in AegisNode is opt-in. It changes middleware behavior for chosen apps, then auth decides how clients prove identity. Swagger documents what you expose.

Settings

API Apps

Mark specific app names as API apps when you want JSON-first behavior. This does not generate routes; it only applies API middleware to the app mount declared in settings.apps. 

export default {
  apps: [
    { name: 'forums', mount: '/forums' },
    { name: 'threads', mount: '/api/threads' },
    { name: 'users', mount: '/users' },
  ],
  api: {
    apps: ['threads'],
    disableCsrf: true,
    requireJsonForUnsafeMethods: true,
    noStoreHeaders: true,
  },
};
// routes.js
import threadsApi from './apps/threads/routes.js';

export default {
  register(route) {
    route.use('/api/threads', threadsApi);
  },
};

Settings

Auth, JWT, And OAuth2

Use JWT when the forum API is mainly consumed by your own frontend, mobile app, or admin client. Use OAuth2 when third-party apps need standards-based delegated access, client credentials, or authorization-code flows. 

auth: {
  enabled: true,
  provider: 'jwt',
  storage: {
    driver: 'file',
    filePath: 'storage/forum-auth.json',
  },
  jwt: {
    secret: process.env.JWT_SECRET,
    issuer: 'forum',
    audience: 'forum-clients',
    expiresIn: '15m',
    refreshExpiresIn: '7d',
  },
}
// JWT forum API route
route.get('/me/threads', auth.required(), ThreadsView.mine);
auth: {
  enabled: true,
  provider: 'oauth2',
  storage: {
    driver: 'database',
    tableName: 'forum_oauth_store',
  },
  oauth2: {
    grants: ['authorization_code', 'refresh_token', 'client_credentials'],
    requirePkce: true,
    server: {
      enabled: true,
      basePath: '/oauth',
      issuer: 'https://forum.example.com',
      allowHttp: false,
    },
  },
}
Practical rule: choose JWT first for first-party forum clients. Move to OAuth2 when you need external app integrations, delegated scopes, or standards-compliant machine-to-machine access.

Settings

Swagger

Swagger serves the OpenAPI JSON plus a browser UI. This is useful when mobile or frontend teams need a stable reference for the forum threads API. 

swagger: {
  enabled: true,
  docsPath: '/docs',
  jsonPath: '/openapi.json',
  documentPath: 'openapi/forum-api.json',
  explorer: true,
}
{
  "openapi": "3.0.3",
  "info": {
    "title": "Forum Threads API",
    "version": "1.0.0"
  },
  "paths": {
    "/api/threads": {
      "get": {
        "summary": "List threads"
      }
    }
  }
}
