Quick Start

Create the forum app, install dependencies, and get the dev server running fast.

This page is the shortest path from zero to a working forum project. It focuses on commands, generated files, and the first route setup.

Step 1

Create The Project

Start by scaffolding a project named forum and installing dependencies.

npm install -g aegisnode

aegisnode startproject forum
aegisnode startproject forum-ts --typescript
npm --prefix forum install

The scaffold gives you app.js, loader.cjs, settings.js, routes.js, and a generated .env with APP_SECRET. The generated settings.js also embeds the same secret as a fallback literal.

Use --typescript when you want the scaffold to generate app.ts, settings.ts, routes.ts, app *.ts files, and tsconfig.json.

If loader.cjs or the main app entry file is missing later, run aegisnode generateloader --project forum to restore the startup entry files.

Step 2

Generate Forum Apps

Create the main apps used in this documentation.

aegisnode createapp forums --project forum
aegisnode createapp threads --project forum
aegisnode createapp users --project forum
aegisnode doctor --app users --project forum
aegisnode fix --app users --project forum

Each app gets views, models, services, validators, subscribers, routes, an app-local utils.js, and tests. In TypeScript mode those become .ts files, including generated tests like models.test.ts.

Use aegisnode doctor --app <app-name> to check one app for missing scaffold files and missing registration. Use aegisnode fix --app <app-name> to recreate the missing pieces without overwriting files that already exist.

Step 3

Run The Development Server

Use the CLI runserver command in development mode.

aegisnode runserver --project forum
Important: in development use aegisnode runserver. Outside development use node loader.cjs. The framework now enforces this.

Reverse Proxy

Enable Trust Proxy When HTTPS Ends Upstream

If the forum app sits behind Nginx, Apache, Passenger, or another reverse proxy that terminates HTTPS before Node, set top-level trustProxy in settings.js. 

export default {
  trustProxy: 1,
};

This is the AegisNode equivalent of Express app.set('trust proxy', 1). It fixes req.secure, req.protocol, secure cookies, client IP detection, and HTTPS-aware auth checks behind the proxy.

Prefer an exact value such as 1, 'loopback', or a subnet string instead of true.

Step 4

Mount The Forum Apps

Connect the generated apps in the root routes.js.

import forums from './apps/forums/routes.js';
import threads from './apps/threads/routes.js';
import users from './apps/users/routes.js';

export default {
  register(route) {
    route.use('/forums', forums);
    route.use('/threads', threads);
    route.use('/users', users);
  },
};

In a TypeScript project, use the same imports with .ts extensions instead of .js.
