Foundation Settings

Configure the forum app host, port, HTTPS behavior, logging, and environment-specific overrides.

These settings define how the application boots and how it behaves across development, staging, and production.

Settings

Top-Level Settings

The top-level config describes the forum app identity, runtime env, network binding, proxy trust, and declared apps.

export default {
  appName: 'forum',
  env: process.env.NODE_ENV || 'development',
  host: process.env.HOST || '0.0.0.0',
  port: process.env.PORT ? Number(process.env.PORT) : 3000,
  trustProxy: false,
  staticDir: 'public',
  apps: [
    { name: 'forums', mount: '/forums' },
    { name: 'threads', mount: '/threads' },
    { name: 'users', mount: '/users' },
  ],
};

Settings

HTTPS

Use direct HTTPS only when Node itself should terminate TLS. If a proxy or hosting panel handles HTTPS, keep this off and set trustProxy instead.

https: {
  enabled: true,
  keyPath: 'certs/forum-key.pem',
  certPath: 'certs/forum-cert.pem',
  options: {
    minVersion: 'TLSv1.2',
  },
}

Settings

Logging

Control log verbosity globally. For a forum app, a common pattern is verbose logs in development and quieter logs in production.

logging: {
  level: process.env.LOG_LEVEL || 'info',
}

Settings

Environment Overrides

Keep one settings.js and let environments.default plus environments[env] refine config per environment.

environments: {
  default: {
    logging: { level: 'info' },
  },
  development: {
    logging: { level: 'debug' },
  },
  production: {
    host: '127.0.0.1',
    logging: { level: 'warn' },
    trustProxy: 1,
  },
}
