Core Concepts

Understand how a forum app is split into routes, views, validators, services, models, and subscribers.

AegisNode stays readable by keeping responsibilities narrow and predictable. This page covers the file shape and the injected runtime objects used throughout the framework.

Structure

Project Shape

Each generated app follows the same structure. That consistency is what keeps a multi-app forum project understandable.

apps/
  threads/
    views.js
    models.js
    services.js
    utils.js
    validators.js
    subscribers.js
    routes.js
    tests/

If the project was scaffolded with --typescript, the same files are generated as views.ts, models.ts, services.ts, utils.ts, validators.ts, subscribers.ts, and routes.ts.

  • routes.js registers HTTP routes.
  • views.js reads requests and builds responses.
  • validators.js sanitizes and validates input.
  • services.js owns business rules.
  • models.js talks to the data layer.
  • utils.js holds small pure app-local utility functions.
  • subscribers.js registers event listeners.

Runtime

Runtime Injection

Handlers and constructors receive framework services as dependencies instead of importing them from deep internals.

class ThreadsView {
  static async create({ service, validator, auth, mail }, req, res, next) {
    try {
      const payload = validator.create(req.body || {});
      const created = await service.createThread(payload, req.auth?.sub || null);
      await mail.send({
        to: 'mods@example.com',
        subject: 'New thread created',
        text: `Thread: ${created.title}`,
      });
      res.status(201).json({ data: created });
    } catch (error) {
      next(error);
    }
  }
}

You also get request-scoped helpers on req.aegis for translation, auth, mail, events, helpers, and more.

Flow

Default Layer Flow

route -> view -> validator -> service -> model

In the forum app, a thread creation request typically passes through this flow:

// routes.js
route.post('/threads', ThreadsView.create);

// views.js
const payload = validator.create(req.body || {});
const created = await service.createThread(payload, req.auth?.sub);

// services.js
return this.model.create(payload, authorId);

// models.js
return this.db.table('threads').insert([{ ...payload, authorId }]).run();
