Deployment

Run the forum correctly in development and production with the expected AegisNode entrypoint.

Startup mode is now enforced. The framework expects one launcher in development and a different one outside development, so your docs and deploy scripts should match that contract.

Mode

Development Startup

In development, use the CLI server command. It is the only allowed launcher in development mode. 

NODE_ENV=development aegisnode runserver --project forum
Rejected in development: node app.js and node loader.cjs.

Mode

Production Startup

In production or staging, use the generated CommonJS loader. This is the stable entrypoint for process managers and hosting panels. 

NODE_ENV=production node loader.cjs
Rejected outside development: aegisnode runserver.

Hosting

Phusion Passenger And Reverse Proxies

Passenger and similar platforms typically terminate HTTPS before the Node process. In that case, keep direct HTTPS disabled and trust the proxy hop that sits in front of the forum app. 

export default {
  env: 'production',
  host: '127.0.0.1',
  port: 3000,
  trustProxy: 1,
  https: {
    enabled: false,
  },
};

Project Scripts

Recommended Scripts

Keep the project scripts aligned with the enforced launcher rules.

{
  "scripts": {
    "dev": "aegisnode runserver",
    "start": "node loader.cjs"
  }
}

If loader.cjs is missing, restore the production entry files with aegisnode generateloader. The doctor command also flags a missing loader.cjs when the resolved app env is production.
