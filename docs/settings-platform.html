Platform Settings

Control how the forum runs in production: protection, persistence, uploads, real-time events, and email.

These settings shape the runtime platform around the forum app. Use them to decide how the app behaves when traffic spikes, how files are stored, how messages are delivered, and how maintenance windows are handled.

Feature

Maintenance Mode

Maintenance mode is useful when the forum needs a schema migration or a planned deployment window. You can show a custom route first and fall back to the built-in maintenance view when that route is missing. 

export default {
  maintenance: {
    enabled: true,
    route: '/maintenance',
    excludePaths: ['/health', '/openapi.json'],
    retryAfter: 600,
  },
};
// routes.js
route.get('/maintenance', (req, res) => {
  res.status(503).render('maintenance', {
    title: 'Forum upgrade in progress',
    eta: '10 minutes',
  });
});

Settings

Security

The security block bundles Helmet headers, CSP, rate limiting, and CSRF settings. For a forum app, that usually means allowing your CDN, limiting burst traffic, and protecting write actions such as creating threads or replies. 

security: {
  appSecret: process.env.APP_SECRET,
  headers: {
    enabled: true,
    csp: {
      directives: {
        scriptSrc: ["'self'", 'https://cdn.jsdelivr.net'],
        styleSrc: ["'self'", "'unsafe-inline'", 'https://fonts.googleapis.com'],
        fontSrc: ["'self'", 'https://fonts.gstatic.com'],
        connectSrc: ["'self'", 'wss://forum.example.com'],
      },
    },
  },
  ddos: {
    windowMs: 60000,
    maxRequests: 120,
    skipPaths: ['/health'],
  },
  csrf: {
    enabled: true,
    headerName: 'x-csrf-token',
    sameSite: 'lax',
  },
}
If your forum sits behind Nginx or Passenger, set top-level trustProxy so rate limiting and secure cookies use the real client address and protocol.

Settings

Database

Use the database block to connect the forum to Postgres, MySQL, SQLite, or Mongo. The example below uses Postgres for threads, replies, and user records. 

database: {
  enabled: true,
  dialect: 'pg',
  config: {
    host: process.env.DB_HOST || '127.0.0.1',
    port: process.env.DB_PORT ? Number(process.env.DB_PORT) : 5432,
    database: process.env.DB_NAME || 'forum',
    user: process.env.DB_USER || 'forum',
    password: process.env.DB_PASSWORD || '',
  },
}
class ThreadsModel {
  constructor({ dbClient }) {
    this.db = dbClient;
  }

  latest(limit = 20) {
    return this.db('threads').select('*').orderBy('updated_at', 'desc').limit(limit);
  }
}

Settings

Cache

Cache is useful for forum home-page summaries, category counts, and other repeated reads that do not need a fresh database query every time. 

cache: {
  enabled: true,
  driver: 'memory',
  options: {},
}
class ForumsService {
  constructor({ cache, model }) {
    this.cache = cache;
    this.model = model;
  }

  async stats() {
    const key = 'forums:stats';
    const cached = await this.cache.get(key);
    if (cached) return cached;

    const stats = await this.model.stats();
    await this.cache.set(key, stats, 60);
    return stats;
  }
}

Settings

WebSocket

Enable WebSocket support when the forum needs live thread updates, moderation notifications, or typing indicators. The built-in runtime uses Socket.IO. 

websocket: {
  enabled: true,
  cors: {
    origin: ['https://forum.example.com'],
    credentials: true,
  },
}
class ThreadsSubscriber {
  constructor({ events, io }) {
    this.events = events;
    this.io = io;
  }

  register() {
    this.events.on('threads.created', (thread) => {
      this.io.emit('threads:created', thread);
    });
  }
}

Settings

Uploads

Use uploads for forum attachments such as screenshots, PDFs, or avatar images. The config controls file limits, MIME checks, and where uploaded files are written. 

uploads: {
  enabled: true,
  dir: 'storage/uploads',
  maxFileSize: '10mb',
  maxFiles: 3,
  allowedMimeTypes: ['image/jpeg', 'image/png', 'application/pdf'],
  allowedExtensions: ['.jpg', '.jpeg', '.png', '.pdf'],
}
// routes.js
route.upload('post', '/threads/:id/attachments', {
  fieldName: 'attachments',
  maxFiles: 3,
}, ThreadsView.attachFiles);

Settings

Mail

Mail covers forum notifications such as welcome emails, password reset flows, and moderator alerts. The built-in mail manager uses Nodemailer and is available through dependency injection. 

mail: {
  enabled: true,
  defaults: {
    from: 'Forum <noreply@forum.example.com>',
    replyTo: 'support@forum.example.com',
  },
  transport: {
    host: process.env.SMTP_HOST || '127.0.0.1',
    port: process.env.SMTP_PORT ? Number(process.env.SMTP_PORT) : 1025,
    secure: false,
  },
}
class UsersService {
  constructor({ mail, i18n }) {
    this.mail = mail;
    this.i18n = i18n;
  }

  async sendWelcome(user) {
    await this.mail.send({
      to: user.email,
      subject: this.i18n.t('mail.welcome.subject'),
      text: `Welcome to the forum, ${user.displayName}.`,
    });
  }
}
