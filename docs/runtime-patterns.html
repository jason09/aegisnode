Runtime Patterns

Use consistent request flow, validation, and event wiring as the forum grows.

These patterns are where the framework’s structure becomes practical. They help keep forum code maintainable when thread creation, moderation, uploads, auth, and notifications all start interacting.

Pattern

Middleware

Put cross-cutting concerns in middleware instead of pushing them into views. In a forum app this usually means session hydration, request timing, moderation flags, or tenant resolution. 

export default function forumContext(req, res, next) {
  res.locals.currentUser = req.auth || null;
  res.locals.isModerator = Boolean(req.auth?.roles?.includes('moderator'));
  next();
}
// routes.js
import forumContext from './middleware/forum-context.js';

export default {
  register(route) {
    route.use(forumContext);
    route.use('/threads', threadsRoutes);
  },
};

Middleware can also receive injected runtime context. That is useful when the forum middleware needs translation, logging, helpers, or auth utilities without importing framework internals. 

async function attachForumLocals({ helpers, i18n, logger }, req, res, next) {
  logger.info(`Forum request: ${req.method} ${req.originalUrl}`);
  res.locals.t = i18n.t;
  res.locals.formatNumber = helpers.number;
  next();
}

export default {
  register(route) {
    const authGuard = (req, res, next) => req.aegis.auth.middleware()(req, res, next);

    route.use(attachForumLocals);
    route.get('/account', authGuard, UsersView.account);
  },
};
In AegisNode, a 4-argument middleware function means (context, req, res, next), not Express error middleware. Do not use (err, req, res, next) in route chains.

Pattern

Validators

Validators sit between the route/view layer and the service layer. They are the right place to normalize and reject thread payloads before business logic runs. 

class ThreadsValidator {
  constructor({ i18n }) {
    this.i18n = i18n;
  }

  create(input = {}) {
    if (!input.title || input.title.trim().length < 5) {
      throw new Error(this.i18n.t('threads.validation.title_short'));
    }

    return {
      title: input.title.trim(),
      body: String(input.body || '').trim(),
      forumId: Number(input.forumId),
    };
  }
}

Pattern

Strict Layers

Strict layers are most useful when the codebase gets large enough that shortcuts start to pile up. They keep the forum flow explicit: views do request work, services do business rules, and models do persistence. 

architecture: {
  strictLayers: true,
}
// good
const payload = validator.create(req.body);
const thread = await service.createThread(payload, req.auth?.sub);

// avoid
await model.create(req.body, req.auth?.sub);

Pattern

Subscribers

Subscribers react to events after the main request flow succeeds. In the forum app, that is a clean place for activity logs, websocket pushes, and moderator notifications. 

class ThreadsSubscriber {
  constructor({ events, mail, i18n }) {
    this.events = events;
    this.mail = mail;
    this.i18n = i18n;
  }

  register() {
    this.events.on('threads.created', async (thread) => {
      await this.mail.send({
        to: 'mods@forum.example.com',
        subject: this.i18n.t('mail.moderation.subject'),
        text: `New thread awaiting review: ${thread.title}`,
      });
    });
  }
}
