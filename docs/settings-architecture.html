Architecture Settings

Decide how the forum project is mounted, initialized, and kept disciplined as it grows.

These settings are less about user-facing features and more about project structure. They matter once the forum has multiple apps, startup hooks, and a team that needs predictable boundaries.

Settings

Architecture

The architecture block currently centers on strict layer enforcement. It helps stop a forum view from bypassing validation or a service from turning into a second controller layer. 

architecture: {
  strictLayers: true,
}
// allowed
route.post('/threads', ThreadsView.create);

// expected internal flow
view -> validator -> service -> model

Settings

Auto Mount Apps

Enable auto-mounting when the forum should wire each app automatically from settings.apps. Leave it off when you want a central routes.js to control mount order and wrappers. 

autoMountApps: true,
apps: [
  { name: 'forums', mount: '/forums' },
  { name: 'threads', mount: '/threads' },
  { name: 'users', mount: '/users' },
]
Use autoMountApps: false if the root routes file needs to wrap some apps with extra middleware or mount API and web variants differently.

Settings

Loaders

Loaders run during startup before routes mount. In a forum app they are a good place to warm caches, seed default roles, or connect background jobs. 

loaders: [
  async ({ logger, cache }) => {
    logger.info('warming forum cache');
    await cache.set('forum:booted', true, 30);
  },
  { path: 'loaders/seed-roles.js', options: { roles: ['admin', 'moderator'] } },
]
// loaders/seed-roles.js
export default async function seedRoles({ logger, options, services }) {
  await services.forApp('users').createDefaultRoles(options.roles);
  logger.info('forum roles ready');
}

Settings

Apps

The apps array declares which sub-apps exist and where they mount. This is the backbone of a modular forum, because routes, API behavior, and template bases all reference app names from this list. 

apps: [
  { name: 'forums', mount: '/forums' },
  { name: 'threads', mount: '/threads' },
  { name: 'users', mount: '/users' },
]
// shorthand form when mount should be /name
apps: ['forums', 'threads', 'users']
